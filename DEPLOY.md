# Deploying FemboyChat (Cloudflare Pages + Supabase)

This project is a Vite + React SPA. It runs out of the box in **demo mode**
(everything in the browser), and upgrades to a real, cross-device backend when
Supabase credentials are present.

## 1. Cloudflare Pages

In the Cloudflare Pages "Set up builds and deployments" screen for this repo:

| Field | Value |
| --- | --- |
| Framework preset | **Vite** |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Production branch | `main` |

Then **Save and Deploy**. The site publishes to `*.pages.dev`.

> The Vite `base` is `/` (see `vite.config.ts`), which is required for Cloudflare
> Pages and custom domains. (The old `/FemboyChat/` base only worked for GitHub
> Pages project sites.)

### Custom domain (e.g. from reg.ru)

1. In Cloudflare, **Add a site** for your domain and, at reg.ru, change the
   domain's nameservers to the two Cloudflare gives you.
2. In the Pages project → **Custom domains** → add your domain.
3. Cloudflare issues the TLS certificate automatically.

## 2. Supabase backend (real accounts + cross-device + FemboyAI)

Production builds already point at the provisioned Supabase project
(`azriyxvofeceosuoptcm`) via a browser-safe anon key baked into the bundle, so
the deployed site uses the real backend automatically. To point at a different
project instead, set `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` in the
Cloudflare Pages **Environment variables** and redeploy.

The database schema, RLS, realtime, RPCs, seeded community, onboarding trigger
and the FemboyAI bot are in [`supabase/schema.sql`](./supabase/schema.sql).

### Email login (one Supabase setting)

FemboyChat signs users in with an emailed **6-digit code** (Email OTP). In
Supabase → **Authentication → Providers → Email**, make sure email is enabled,
and in **Authentication → Email Templates → Magic Link**, include the code token
so the email actually shows it, e.g.:

```
Твой код для входа в FemboyChat: {{ .Token }}
```

(Supabase's built-in email has low rate limits; for volume add your own SMTP in
Auth settings.) Without this, users receive a link instead of a code.

## 3. FemboyAI (the assistant bot)

Every user automatically gets a chat with **FemboyAI**. Replies are generated by
the `femboy-ai-fc` Edge Function (see `supabase/functions/femboy-ai-fc`), which
is triggered on new messages and reads its model key from Supabase Vault.

It is provider-agnostic; add **one** of these keys as a Vault secret to enable
real answers (otherwise the bot uses a cute canned fallback):

```sql
-- run once in the Supabase SQL editor (or ask the agent to do it):
select vault.create_secret('YOUR_KEY', 'GEMINI_API_KEY', 'FemboyAI model key');
-- or GROQ_API_KEY / OPENROUTER_API_KEY / GITHUB_MODELS_TOKEN
```

> Note: GitHub Models is being retired by GitHub (its API now returns a
> "retirement brownout" error), so prefer **Google Gemini** (free tier) or
> **Groq** (free tier).
